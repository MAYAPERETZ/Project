MARS - RISCV Assembly and Runtime Simulator

Release 1.0

September 2019

Operating Requirements

MARS is written in Java and requires release 9 or later of the J2SE Java Runtime Environment (JRE) to work. The graphical IDE is implemented using Swing. It has been tested only in Windows 10.

Some MARS Assembler and Simulator Limitations

Releases 1.0 assemble and simulate nearly all the RISCV32 and RISCV64 instructions documented in the textbook The RISC-V Reader: An Open Architecture Atlas by David Patterson and Andrew Waterman , 2017.

Limitations of MARS as of Release 1.0 include:

  • The limitation of the RISCV version probably includes all MIPS version latest version limitations.
    • For further details about MIPS latest version limitation, see the relevant page in: http://www.cs.missouristate.edu/MARS/.
  • TODO:The CSRs window implementation remained almost untouched. This means that the current implementation doesn't fit the RISCV architecture. This window was not removed in order to future integration into RISCV architecture
  • TODO:The current implementation was tested a lot, but yet not completely. There may be some instruction which will have bugs
  • TODO:The macros and the dump files were not modified according to the RISCV architecture. The implementation of MIPS remained as is and there its behavior in the new RISCV version is unexpected
  • Bug:The FCSR is missing the inexact flag. This is because the implementation of it using Java's floating point types is not possible, since Java's floating-point numbers are not perfectly accurate
